# Contributing to Awesome Timeline Explorers

We welcome contributions to this curated list of interactive historical web explorers! Your help ensures that this resource remains up-to-date, comprehensive, and valuable for enthusiasts, researchers, and anyone looking to explore the past in an engaging way.

## How to Contribute

There are several ways you can contribute to this project:

### 1. Suggesting New Sites

If you've found an amazing interactive historical web explorer that isn't on our list, please suggest it!

*   **Open an Issue:** The simplest way is to [open a new issue](https://github.com/your-username/awesome-timeline-explorers/issues/new?assignees=&labels=enhancement&projects=&template=site_suggestion.md&title=Site+Suggestion%3A+) and provide the details of the site.
*   **Submit a Pull Request (Preferred):** For more direct contributions, you can add the site yourself by following these steps:

    1.  **Fork** this repository.
    2.  **Clone** your forked repository to your local machine:
        `git clone https://github.com/your-username/awesome-timeline-explorers.git`
    3.  **Create a new branch** for your contribution:
        `git checkout -b feature/add-new-site-name` (e.g., `git checkout -b feature/add-radiooooo`)
    4.  **Edit the `README.md` file**: Add your site to the appropriate category. If a suitable category doesn't exist, consider creating a new one or discussing it in an issue first.
        *   **Formatting:** Please follow the existing format: `- **Site Name** ([URL]): Brief description and fit justification [web:X].`
        *   **Source Citation:** Ensure you include a `[web:X]` citation, referring to a credible source (e.g., Wikipedia, official announcement, reputable tech blog) that verifies the site's purpose and features. If possible, provide a link to the source at the bottom of the `README.md`.
    5.  **Ensure all criteria are met:** Double-check that the site you're adding meets all the [Criteria for Inclusion](#criteria-for-inclusion) listed below.
    6.  **Commit your changes** with a clear and concise commit message (e.g., `feat: Add Radiooooo to Media Archives`).
    7.  **Push your branch** to your forked repository:
        `git push origin feature/add-new-site-name`
    8.  **Open a Pull Request** against the `main` branch of this repository. Provide a brief explanation of your changes and why the site is a good fit.

### 2. Improving Existing Entries

If you find an error in an existing entry, think a description could be improved, or discover a broken link, please submit a Pull Request or open an issue.

### 3. General Improvements

Suggestions for improving the structure, criteria, or any other aspect of this list are always welcome. Please [open an issue](https://github.com/your-username/awesome-timeline-explorers/issues/new?assignees=&labels=enhancement&projects=&template=feature_request.md&title=Feat%3A+) to discuss your ideas.

## Criteria for Inclusion

Sites must be:

-   **Live and Accessible**: Fully functional with no major technical issues.
-   **Free Access**: No subscription required or paywalls.
-   **Interactive**: Feature timelines, sliders, overlays, or era-specific interfaces that encourage dynamic exploration.
-   **Serendipitous Discovery**: Designed for browsing and exploration, rather than requiring direct search for specific items.
-   **Authentic Experience**: Recreate historical or nostalgic interfaces accurately and immersively.
-   **Community-Driven (Preferred)**: Utilize public domain content or allow for community contributions.

## Code of Conduct

We aim to foster an open and welcoming environment. Please be respectful and constructive in all your interactions. Harassment of any kind will not be tolerated.
