# Contributing to Awesome Timeline Explorers [![Awesome](https://awesome.re/badge.svg)](https://awesome.re)

We welcome contributions to this curated list of interactive historical web explorers! Your help ensures that this resource remains up-to-date, comprehensive, and valuable for enthusiasts, researchers, and anyone looking to explore the past in an engaging way.

## How to Contribute

There are several ways you can contribute to this project:

### 1. Suggesting New Sites

If you've found an amazing interactive historical web explorer that isn't on our list, please suggest it!

Open an issue or submit a pull request. The simplest way is to open a new issue and provide the details of the site. For more direct contributions, you can add the site yourself by following these steps:

1. Fork this repository.
2. Clone your forked repository to your local machine: `git clone https://github.com/your-username/awesome-timeline-explorers.git`
3. Create a new branch for your contribution: `git checkout -b feature/add-new-site-name` (e.g., `git checkout -b feature/add-radiooooo`)
4. Edit the `README.md` file: Add your site to the appropriate category. If a suitable category doesn't exist, consider creating a new one or discussing it in an issue first. Follow the existing format: `- **Site Name** ([URL]): Brief description and fit justification [web:X].` Ensure you include a `[web:X]` citation, referring to a credible source (e.g., Wikipedia, official announcement, reputable tech blog) that verifies the site's purpose and features. If possible, provide a link to the source at the bottom of the `README.md`.
5. Ensure all criteria are met: Double-check that the site you're adding meets all the criteria for inclusion listed below.
6. Commit your changes with a clear and concise commit message (e.g., `feat: Add Radiooooo to Media Archives`).
7. Push your branch to your forked repository: `git push origin feature/add-new-site-name`
8. Open a pull request against the `main` branch of this repository. Provide a brief explanation of your changes and why the site is a good fit.

### 2. Improving Existing Entries

If you find an error in an existing entry, think a description could be improved, or discover a broken link, please submit a pull request or open an issue.

### 3. General Improvements

Suggestions for improving the structure, criteria, or any other aspect of this list are always welcome. Please open an issue to discuss your ideas.

## Criteria for Inclusion

Sites must be live and accessible, free access, interactive, serendipitous discovery, authentic experience, and community-driven. Specifically: Fully functional with no major technical issues; no subscription required or paywalls; feature timelines, sliders, overlays, or era-specific interfaces that encourage dynamic exploration; designed for browsing and exploration, rather than requiring direct search for specific items; recreate historical or nostalgic interfaces accurately and immersively; and utilize public domain content or allow for community contributions.

## Code of Conduct

We aim to foster an open and welcoming environment. Please be respectful and constructive in all your interactions. Harassment of any kind will not be tolerated.
